CloudSync Component

Компонент для автоматической синхронизации с облачными хранилищами (Google Drive, Dropbox)
1. Описание проекта

Проект представляет собой кроссплатформенный внешний компонент на C++, предназначенный для автоматической синхронизации локальных директорий с облачными хранилищами. Основная задача — обеспечить надежную передачу файлов, отслеживание изменений и обработку конфликтов версий.
2. Основная идея и функционал

    Абстракция провайдера: Единый интерфейс для работы с разными облаками (Google Drive API, Dropbox API).

    Автоматическое отслеживание: Мониторинг локальной папки на предмет появления, изменения или удаления файлов.

    Фоновая синхронизация: Выполнение операций загрузки/выгрузки без блокировки основного потока приложения.

    Юнит-тестирование: Полное покрытие логики работы с файловой системой и формирования API-запросов.

3. План реализации

    Проектирование: Определение интерфейсов ICloudProvider и схем аутентификации (OAuth2).

    Разработка ядра: Реализация логики сравнения локальных и облачных метаданных.

    Интеграция API: Написание конкретных реализаций для Google Drive/Dropbox с использованием libcurl.

    Создание обвязки: Оформление проекта как ВК (внешнего компонента).

    Тестирование: Написание тестов для имитации сетевых ответов и проверки целостности файлов.

4. Архитектура системы (UML описание)

Проект строится на следующих ключевых классах:

    SyncManager: Главный контроллер, управляющий очередью задач.

    FileWatcher: Модуль, отслеживающий события в ОС (создание/изменение файла).

    ICloudProvider (Интерфейс): Содержит методы upload(), download(), list(), remove().

    GDriveProvider / DropboxProvider: Конкретные реализации API.

5. Технические требования

    Сборка: CMake 3.10+.

    Язык: C++17 (или выше).

    Зависимости: libcurl, OpenSSL, nlohmann/json (для парсинга ответов API).

    Кроссплатформенность: Поддержка Windows и Linux.
